# Autobot

## Лицензирование без хранения списка ключей

Приложение **не хранит таблицу со всеми валидными ключами**.
Ключ проверяется алгоритмически по формату и контрольной сумме.

Формат ключа:

`FBOT-<7|14|30>-<B5>-<B8>-<B3>-<B9>-<CS>`

Где:
- `7|14|30` — срок в днях,
- `B5` — 4-значный блок, кратный 5,
- `B8` — 4-значный блок, кратный 8,
- `B3` — 4-значный блок, кратный 3,
- `B9` — 4-значный блок, кратный 9,
- `CS` — контрольная сумма (2 hex символа).

В БД `licenses.db` хранится текущее состояние активации и hash уже использованных ключей.

### Привязка к ПК при первой активации

При первой активации ключа программа сохраняет отпечаток железа:
- имя компьютера,
- MAC-адрес (в виде hash).

На каждом следующем запуске выполняется проверка совпадения. Если приложение запущено на другом ПК (не совпадает имя и/или MAC), лицензия считается недействительной.

Каждый ключ **одноразовый**: повторная активация того же ключа будет отклонена.

## Генерация ключей

Ключи генерируются в `key_generator.py`.

Примеры:

```bash
python key_generator.py --days 7
python key_generator.py --days 14 --count 3
python key_generator.py --days 30 --count 10
```

## Сборка в единый `.exe` через Nuitka

Добавлен GitHub Actions workflow `.github/workflows/build-windows-exe.yml`, который собирает **единый exe-файл** (`launcher.exe`) через Nuitka в режиме `--onefile`.

В сборку автоматически включаются все нужные ресурсы:
- звуки,
- картинки,
- шаблоны,
- `config.json`.

### Как получить готовый exe

1. Откройте вкладку **Actions** в GitHub.
2. Запустите workflow **Build Windows EXE** (кнопка **Run workflow**).
3. После завершения скачайте artifact **launcher-windows**.
4. Распакуйте архив `launcher-windows.zip` — внутри будет `launcher.exe` для пользователей без Python.
